## Creating Custom Visualizations

### Supported Panel Types

| Panel Type | Use Case | Best For |
|-----------|----------|----------|
| **Stat** | Single metric display | KPI values, totals, percentages |
| **Pie Chart** | Proportion breakdown | Memory by type/sensitivity, distribution |
| **Bar Gauge** | Comparison of values | Operation counts, rate limits |
| **Time Series** | Trend over time | Growth, activity patterns, trends |
| **Table** | Structured data | Policy list, agent registry, audit logs |
| **Heatmap** | Time-based distribution | Activity patterns by hour/day |

### Example Query: Custom Agent Metrics

```json
{
  "dashboard": {
    "title": "Agent-Specific Dashboard",
    "panels": [
      {
        "title": "Agent Performance",
        "type": "stat",
        "targets": [
          {
            "url": "/stats/agent-activity",
            "filter": "?agent_id=agent-001"
          }
        ]
      },
      {
        "title": "Memory Usage by Agent",
        "type": "piechart",
        "targets": [
          {
            "url": "/stats/memory-summary",
            "filter": "?scope=agent"
          }
        ]
      }
    ]
  }
}
```

## Alert Examples

### Alert Rule 1: High Memory Growth
```
IF:   total_items > 5000000
THEN: Send notification to ops-team@company.com
      "High memory growth detected. Current: {value}"
      
FOR:  5 minutes (sustained, not spike)
```

### Alert Rule 2: Certificate Expiry
```
IF:   days_remaining < 7
THEN: Send notification to security@company.com
      "SSL certificate expires in {days_remaining} days"
      
FOR:  1 hour (once per check, no spam)
```

### Alert Rule 3: Rate Limiting Active
```
IF:   daily_hits > 100
THEN: Send notification to platform@company.com
      "{violator_count} agents hitting rate limits: {limit_types}"
      
FOR:  10 minutes (check for sustained issues)
```

## Performance Tips

### Query Optimization
1. **Use date range filters** to limit data volume
2. **Filter by agent_id** for single-agent analysis
3. **Aggregate at query level** (sum, avg, max) rather than in visualization
4. **Cache policy responses** (change infrequently)

### Dashboard Design
1. **Keep dashboards focused** (max 5-6 panels per dashboard)
2. **Use appropriate visualizations** (pie for parts-of-whole, time series for trends)
3. **Set reasonable refresh rates** (don't refresh faster than needed)
4. **Use dashboard variables** for filtering without multiple queries

### Storage
- **Datasource caching**: Enabled for 60s
- **Memory snapshots**: Taken every 5 minutes
- **Historical data**: Retained based on TTL policies
